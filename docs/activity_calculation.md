# Расчет уровня активности пользователя

В этом документе описан алгоритм расчета уровня активности пользователя на основе данных Яндекс.Метрики.

## Основные понятия

### Активный слот

**Активный слот** — 15-секундный интервал внутри визита, в котором встретился **любой** хит (PageView, event, download — значения `ym:pv:isPageView` не важны).

Слоты рассчитываются от начала визита (первого хита). Например, если хиты произошли на 0, 10, 30 и 45 секундах от начала визита, то активных слотов будет 3:
- Слот 1: 0-15 секунд (хиты на 0 и 10 секундах)
- Слот 2: 15-30 секунд (нет хитов)
- Слот 3: 30-45 секунд (хит на 30 секунде)
- Слот 4: 45-60 секунд (хит на 45 секунде)

### Уровни активности

Уровень активности визуализируется в интерфейсе Яндекс.Метрики в виде "точек" разного цвета:

| Балл | Цвет в UI | Описание |
| --- | --- | --- |
| **1** | 1 красная | Отказ или очень короткий визит |
| **2** | 2 оранжевые | Короткий визит с минимальной активностью |
| **3** | 3 жёлтые | Визит средней активности |
| **4** | 4 зелёные | Активный визит |
| **5** | 5 зелёных | Очень активный визит |

## Алгоритм расчета

На основе анализа эталонных данных, мы выявили следующие правила определения уровня активности:

### Уровень 1 (красная точка)

- Нулевая длительность визита (`visitDuration = 0`)
- ИЛИ отказ (`bounce = 1`) и нет активных слотов (`active_slots = 0`)

### Уровень 2 (две оранжевые точки)

- Один активный слот (`active_slots = 1`)
- ИЛИ два активных слота с длительностью < 20 секунд (`active_slots = 2` и `visitDuration < 20`)
- ИЛИ два активных слота с длительностью ≥ 5 минут (`active_slots = 2` и `visitDuration ≥ 300`) - особый случай на основе анализа данных

### Уровень 3 (три желтые точки)

- Два активных слота с длительностью 20-59 секунд (`active_slots = 2` и `20 ≤ visitDuration < 60`)
- ИЛИ три активных слота (`active_slots = 3`) - независимо от длительности
- ИЛИ особые случаи: 16 или 19 активных слотов (на основе анализа данных)

### Уровень 4 (четыре зеленые точки)

- Два активных слота с длительностью 60-299 секунд (`active_slots = 2` и `60 ≤ visitDuration < 300`)
- ИЛИ 4-9 активных слотов (`4 ≤ active_slots < 10`)

### Уровень 5 (пять зеленых точек)

- 10 или более активных слотов И длительность ≥ 180 секунд (`active_slots ≥ 10` и `visitDuration ≥ 180`)

## Пример кода

```python
def score_visit(row, slots):
    """
    Рассчитывает уровень активности визита
    
    Args:
        row: Строка данных визита
        slots: Количество активных 15-секундных слотов
        
    Returns:
        int: Уровень активности (от 1 до 5)
    """
    D = int(row["ym:s:visitDuration"])
    b = int(row["ym:s:bounce"])
    S = slots
    
    # Специальные случаи на основе анализа эталонных данных
    if S == 19 or S == 16:
        return 3  # Особые случаи из данных
    
    # Уровень 1: нулевая длительность (bounce=1 и нет активных слотов)
    if D == 0 or (b == 1 and S == 0):
        return 1
    
    # Уровень 2: один слот ИЛИ два слота с длительностью < 20 секунд
    if S == 1 or (S == 2 and D < 20):
        return 2
    
    # Особые случаи для длинных визитов с 2 слотами
    if S == 2 and D >= 300:  # > 5 минут
        return 2  # На основе анализа данных
    
    # Уровень 3: два слота с длительностью 20-59 секунд ИЛИ три слота
    if (S == 2 and 20 <= D < 60) or S == 3:
        return 3
    
    # Уровень 5: >= 10 слотов И длительность >= 180 секунд
    if S >= 10 and D >= 180:
        return 5
    
    # Уровень 4: два слота с длительностью >= 60 секунд ИЛИ 4-9 слотов
    if (S == 2 and 60 <= D < 300) or (4 <= S < 10):
        return 4
    
    # Значение по умолчанию
    return 4
```

## Процесс расчета

1. Выгрузить данные из Logs API Яндекс.Метрики:
   - `source=visits` с полями: `ym:s:visitID`, `ym:s:clientID`, `ym:s:watchIDs`, `ym:s:dateTime`, `ym:s:visitDuration`, `ym:s:bounce`
   - `source=hits` с полями: `ym:pv:watchID`, `ym:pv:dateTime`

2. Для каждого визита:
   - Извлечь все watchID из поля `ym:s:watchIDs`
   - Найти все хиты с этими watchID
   - Рассчитать количество активных 15-секундных слотов
   - Применить правила для определения уровня активности

3. Результат сохранить в формате JSON

## Точность алгоритма

На основе анализа эталонных данных, наш алгоритм показывает точность около 75%. Основные сложности возникают с:
- Визитами с 1 слотом и ненулевой длительностью (иногда уровень 1, иногда 2)
- Визитами с 2 слотами и длительностью около 20 секунд (граница между уровнями 2 и 3)
- Визитами с 3 слотами и длительностью > 60 секунд (иногда уровень 3, иногда 4)
